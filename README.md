# MedKit_HTML（仮称）

オフラインで動く、HTMLだけの医療系便利ツール集。
ブラウザでファイルをダブルクリックするだけで動く「置き薬」のようなツール箱を目指す。

> フォルダ名・サイト名は仮称。設計図の壁打ちで正式決定する（`設計図.md` 参照）。

---

## このプロジェクトの絶対ルール

1. **HTMLで作る（超重要）** — ツールはHTML+CSS+JSを1つの `.html` に内包
2. **完全オフライン動作** — CDN・外部フォント・外部API・fetch一切なし。`file://` で開いてそのまま動く
3. **ビルド不要** — Vite / npm / フレームワーク等は使わない。シンプルなHTML/CSS/JSのみ
4. **APIキーをコードに書かない** — そもそもAPIを使わない設計にする（セキュリティ宣言）
5. **レスポンシブWebデザイン** — 職場PCでもスマホでも使える
6. **医療情報の扱い** — 計算ツールは「参考値。最終判断は必ず添付文書・ガイドラインで確認」の注意書きを必ず表示。患者個人情報は入力させない・保存しない

## 使い方

1. `index.html` をブラウザで開く（ダブルクリックでOK。ネット接続・インストール不要）
2. 上部タブで「**腎機能**」「**検査値**」「**輸液**」を切り替える（ツール名・ブラウザタイトルもタブに連動して切り替わる）
3. 各カードに数値を入力し「計算」ボタンを押す（Enterキーでも可）
4. 「リセット」でそのカード（腎機能タブは全体）の入力・結果をクリア
5. 各カードの「**▸ 計算式を見る**」をクリックすると、その計算の根拠式が開閉表示される

**腎機能タブ 結果の読みかた**
- **標準化**（mL/min/1.73m²）→ CKD重症度分類（G1〜G5バッジ）の判定に使う
- **個別化**（mL/min）→ 薬剤の投与量設計に使う。腎排泄型薬剤の減量判断はこちら
- 添付文書がCCr基準の薬剤（例: 新規経口抗凝固薬など）は個別化eCCr、eGFR基準の薬剤は個別化eGFRを見る

## 実装している計算式

### 腎機能タブ

| 項目 | 式 |
|---|---|
| eGFRcr（標準化） | JSN 2008 日本人GFR推算式: 194 × Cr⁻¹·⁰⁹⁴ × 年齢⁻⁰·²⁸⁷（女性 × 0.739） |
| 個別化eGFR | 標準化eGFR × BSA / 1.73 |
| eCCr（個別化） | Cockcroft-Gault式: (140−年齢) × 体重 / (72 × Cr)（女性 × 0.85） |
| 標準化eCCr | eCCr × 1.73 / BSA |
| eGFRcys（標準化） | 日本人向け式: (104 × CysC⁻¹·⁰¹⁹ × 0.996^年齢 [× 0.929 女性]) − 8 |
| 体表面積 (BSA) | Du Bois式: 0.007184 × 身長^0.725 × 体重^0.425 |
| 理想体重 | BMI 22 基準: 22 × 身長(m)² |

参考サイト: 日本腎臓病薬物療法学会「eGFR・eCCrの計算」 https://jsnp.org/egfr/

### 検査値タブ

| 計算 | 式 |
|---|---|
| 補正Ca（Payne式） | 実測Ca + (4 − Alb) |
| mEq ⇔ mg 換算 | mEq = mg ÷ 原子量 × 価数（Na/K/Ca/Mg/Cl のプリセット） |
| Na補正速度チェック | \|目標Na − 現在Na\| ÷ 予定時間。24h換算し目安（10〜12mEq/L/24h）と照合 |
| 水分欠乏量（高Na血症時） | 体重 × 体内水分比 × (現在Na ÷ 140 − 1) |
| 血漿浸透圧（概算） | 2×Na + 血糖/18 + BUN/2.8（正常域目安 275〜295 mOsm/kg） |
| TSAT | 血清鉄 ÷ TIBC × 100（%）。フェリチン併用で鉄欠乏の目安判定 |

### 輸液タブ

| 計算 | 式 |
|---|---|
| 点滴速度 | 速度(mL/h) = 総量 ÷ 時間／滴数(滴/分) = 速度 × 滴数係数 ÷ 60（成人20滴・小児60滴） |
| 希釈・濃度計算 | 濃度(mg/mL) = 薬剤量 ÷ 総量。3項目中2つを入力すると残り1つを算出 |
| 投与可能時間 | 残量 ÷ 流速 |

## 注意事項

- **結果はすべて参考値。** 投与量の最終判断は必ず添付文書・ガイドライン・腎機能別投与量一覧で確認すること
- 本ツールの建付けは**自宅・自己学習用**。業務中の患者対応の判断には使わない（院内リジェクト報告と同じグレーゾーンに戻るため）
- 腎機能の対象は**成人（18歳以上）のみ**。小児には推算式自体が適用できない（入力するとエラー表示）
- eGFRは筋肉量が少ない患者（長期臥床・るい痩・四肢切断など）で高めに出る → シスタチンC併用を検討
- eCCr（CG式）は肥満患者で高めに出る → 理想体重での再計算を検討
- 実測CCrの院内報告値は「mL/min」表記でも体表面積補正済みのことが多い。自施設の表記ルールを確認する
- Na補正速度・TSATの判定閾値は一般的な目安の簡易実装。個別の基準値は必ず一次情報（ガイドライン）で確認する
- 氏名・IDなどの患者個人情報は入力しない（入力欄もない）

## セキュリティ（特典2「セキュリティ宣言」チェック結果）

初回チェック 2026-07-08（eGFR_eCCr）／2026-07-09（Labs_Infusion）。統合版（2026-07-10）は両者のコードをそのまま結合したもので、外部通信・保存要素の追加なし。

| チェック項目 | 結果 |
|---|---|
| APIキーをコードに直接書いていないか | ✅ そもそもAPI・外部サービスを一切不使用 |
| 外部通信（fetch / XMLHttpRequest / CDN / 外部フォント・画像・リンク） | ✅ ゼロ |
| 入力データの保存（localStorage / sessionStorage / Cookie） | ✅ 不使用。閉じれば全消去 |
| 危険なコード実行（eval等） | ✅ 不使用 |
| XSS（入力値を画面に生のまま挿入していないか） | ✅ 入力はparseFloatで数値としてのみ解釈。文字列をDOMに挿入しない |
| 追加の防御 | ✅ CSPメタタグで外部通信・フォーム送信・外部読込をブラウザレベルで遮断 |

→ ネットに公開しても、メールで配布しても安全な構造。ただし**院内PCへの持ち込みは技術ではなく院内セキュリティポリシーの問題**なので、情報システム部門への確認が別途必要。
